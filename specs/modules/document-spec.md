# 模块 Spec：Document（文档管理模块）

> 本模块定义 HR 文档的浏览、上传、管理规范，是知识库的内容入口。仅 HR 角色可上传文档。
>
> 对应变更域：phase-3-user-experience

---

## 1. 范围边界

### 1.1 包含
- 文档列表展示（卡片式布局）
- 文档详情查看（Markdown 渲染）
- 文档分类筛选
- Markdown 文件上传（拖拽 + 点击选择）
- 上传后自动触发分块和索引重建
- 文档元数据管理

### 1.2 不包含
- ❌ 文档分块实现（见 chunk-spec.md）
- ❌ Embedding 生成（见 embedding-spec.md）
- ❌ 向量索引重建（见 vector-spec.md）
- ❌ PDF/Word/Excel 上传（PRD Out of Scope）
- ❌ 文档编辑/删除（扩展点）

## 2. 数据模型

### 2.1 文档模型

```typescript
interface HRDocument {
  id: string;                    // 文档标识，如 "annual_leave"
  filename: string;              // 文件名，如 "年假制度.md"
  title: string;                 // 文档标题
  category: string;              // 分类标识
  categoryName: string;          // 分类中文名
  categoryColor: string;         // 分类颜色
  content: string;               // 完整 Markdown 内容
  chunkCount: number;            // 分块数量
  size: number;                  // 文件大小（字节）
  createdAt: string;             // ISO 时间字符串
  updatedAt: string;             // ISO 时间字符串
}
```

### 2.2 分类模型

```typescript
interface DocumentCategory {
  id: string;                    // 如 "annual_leave"
  name: string;                  // 如 "年假"
  color: string;                 // 如 "#E3F2FD"
  darkColor: string;             // 深色模式颜色，如 "#0D47A1"
  count: number;                 // 该分类下的文档数
}
```

## 3. 后端接口规范

### 3.1 GET /api/documents

详见 [api-spec.md](./api-spec.md)

### 3.2 GET /api/documents/:id

| 属性 | 值 |
|------|-----|
| 路径 | `/api/documents/:id` |
| 方法 | GET |
| 认证 | Bearer JWT |

**响应 200**：
```json
{
  "id": "annual_leave",
  "filename": "年假制度.md",
  "title": "年假制度",
  "category": "annual_leave",
  "categoryName": "年假",
  "content": "# 年假制度\n\n## 年假天数标准\n...",
  "size": 1144,
  "createdAt": "2026-05-17T00:00:00.000Z",
  "updatedAt": "2026-05-17T00:00:00.000Z"
}
```

### 3.3 POST /api/documents/upload

详见 [api-spec.md](./api-spec.md)

### 3.4 上传后处理流程

```
接收 POST /api/documents/upload
    │
    ▼
AuthGuard + RolesGuard（仅 hr 角色）
    │
    ▼
Multer 接收文件
    │
    ▼
文件校验（扩展名 .md、大小 ≤ 1MB）
    │
    ▼
保存到 docs/hr-documents/
    │
    ▼
调用 DocumentLoader 重新加载所有文档
    ├── 解析新文档 → 分块
    ├── 调用 EmbeddingService 生成向量
    └── 存入 VectorStore
    │
    ▼
返回 { filename, chunks, status: 'indexed' }
```

### 4.2 搜索功能

- **位置**：分类筛选标签右侧
- **占位符**："搜索文档标题..."
- **行为**：实时过滤，输入时即时筛选文档列表
- **匹配范围**：仅匹配文档标题（不搜索内容）
- **空结果**：显示"未找到匹配的文档"

### 4.3 文档卡片（DocumentCard）

- **尺寸**：固定宽度 280px，高度自适应
- **背景**：分类色（浅色模式）
- **内容**：
  - 分类标签（小标签，分类色背景）
  - 文档标题（加粗，主文字色）
  - 片段数量（如 "7 个知识片段"）
  - 文件大小（如 "1.1 KB"）
- **交互**：点击打开文档详情
- **悬浮**：轻微上浮 + 阴影加深

### 4.4 分类筛选标签

- **样式**：横向滚动的一组 pill 形状按钮
- **状态**：
  - 未选中：白色背景，灰色边框
  - 选中：主色背景，白色文字
- **选项**：全部、年假、报销、晋升、考勤、福利、自定义
- **计数**：每个分类后显示文档数量（如 "年假 (2)"）

### 4.5 文档详情（DocumentViewer）

- **展示方式**：弹窗（Modal）或侧边抽屉（Drawer）
- **内容**：
  - 顶部：文档标题 + 分类标签 + 关闭按钮
  - 主体：Markdown 渲染内容
  - 底部：文件信息（大小、更新时间）
- **Markdown 渲染**：
  - 标题层级正确渲染
  - 列表缩进正确
  - 代码块有背景色和等宽字体
  - 深色模式适配

### 4.6 上传组件（DocumentUploader）

**触发方式**：
- 点击"上传文档"按钮
- 拖拽文件到页面指定区域

**按钮位置**：页面标题栏右侧，仅 `hr` 角色可见

**上传流程**：
```
选择文件 / 拖拽文件
    │
    ▼
前端校验（仅 .md、≤ 1MB）
    │
    ▼
显示文件信息 + 上传按钮
    │
    ▼
点击上传 → 显示进度条
    │
    ▼
上传成功 → 提示"上传成功，已建立索引" + 刷新列表
    │
    ▼
上传失败 → 显示错误原因
```

**错误提示**：
| 场景 | 提示文案 |
|------|---------|
| 非 .md 文件 | "仅支持 Markdown (.md) 文件" |
| 文件超过 1MB | "文件大小超过 1MB 限制" |
| 上传失败 | "上传失败，请稍后重试" |
| 重复文件名 | "文件已存在，将覆盖原文件" |

### 4.7 文档统计头部

页面标题下方显示知识库统计：
- 总文档数：如 "共 5 个文档"
- 总片段数：如 "32 个知识片段"
- 最后更新时间：如 "最后更新：2026-05-18"

### 4.8 空状态

- **无内置文档**：显示"暂无文档，请联系 HR 上传制度文档"
- **搜索无结果**：显示"未找到匹配的文档，尝试其他关键词"
- **加载中**：卡片区域显示骨架屏

### 4.9 响应式适配

- **桌面端（≥1024px）**：3 列网格
- **平板端（768-1023px）**：2 列网格
- **移动端（<768px）**：单列堆叠

## 5. 权限控制

| 功能 | employee | hr |
|------|----------|-----|
| 查看文档列表 | ✅ | ✅ |
| 查看文档详情 | ✅ | ✅ |
| 分类筛选 | ✅ | ✅ |
| 上传文档 | ❌ | ✅ |
| 查看上传按钮 | ❌ | ✅ |

**前端实现**：根据 `user.role` 条件渲染上传按钮
**后端实现**：`@Roles('hr')` Guard 保护上传接口

## 6. 验收标准

- [ ] 页面展示 5 个内置文档卡片，带分类色块
- [ ] 点击卡片查看完整 Markdown 内容
- [ ] 支持按分类筛选，筛选后仅显示对应分类文档
- [ ] 分类筛选标签显示各分类文档数量
- [ ] employee 账号看不到"上传文档"按钮
- [ ] hr 账号点击"上传文档"，选择 .md 文件，上传成功
- [ ] 上传成功后提示"已建立索引"，文档列表自动刷新
- [ ] 上传非 .md 文件时，前端友好提示错误
- [ ] 上传超过 1MB 文件时，前端友好提示错误
- [ ] 上传新文档后，立即可以针对该文档提问
- [ ] 深色模式下文档卡片和详情页对比度舒适
- [ ] 搜索框可按标题实时过滤文档
- [ ] 搜索无结果时显示友好空状态
- [ ] 页面头部显示文档统计（总数、片段数）
- [ ] 响应式布局：桌面3列 / 平板2列 / 移动端1列

## 7. 与其他模块的关系

```
DocumentModule
    ├── 依赖 AuthModule（鉴权 + 角色控制）
    ├── 依赖 DocumentLoader（解析分块）
    ├── 依赖 EmbeddingService（生成向量）
    ├── 依赖 VectorStore（重建索引）
    └── 提供 UI 给 HR 管理知识库
```
