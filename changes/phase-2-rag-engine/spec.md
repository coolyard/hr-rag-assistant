# 变更域：AI 核心引擎（phase-2-rag-engine）

> 本变更域是项目的核心 AI 能力，包含完整的 RAG Pipeline、多轮对话管理、LLM 生成服务。
> 范围：检索、合并、Prompt 组装、流式生成、对话历史。

---

## 1. 范围边界

### 包含

- 向量检索 + 关键词检索混合算法
- RAG 编排服务（检索 → 合并 → Prompt 组装 → LLM 调用）
- 多轮对话历史管理
- SSE 流式 API 接口
- 前端 SSE 对接与流式渲染

### 不包含

- 用户登录（phase-3）
- Theme 切换（phase-3）
- 文档浏览页面（phase-3）
- 文档上传（phase-3）
- MCP 协议（phase-4）

---

## 2. 依赖的前置域

- phase-1-infrastructure（项目结构、Ollama 连通、真实 Embedding 索引、内置 5 个 HR 文档）

---

## 3. 验收标准

- [ ] 输入"年假怎么请"，返回 Top-3 文档片段，包含来源文件名和相似度分数
- [ ] 输入"公司食堂在哪里"（无关问题），返回空数组（最高相似度 < 0.5）
- [ ] 单轮问答：输入"年假怎么请"，返回基于文档的回答 + 来源引用
- [ ] 多轮问答：追问"那病假呢？"，能承接上文，正确回答病假政策
- [ ] 拒绝场景：输入"张三的工资是多少"，返回 AI-SPEC 定义的拒绝话术
- [ ] 前端输入问题，点击发送，2 秒内出现第一个字
- [ ] 回答以打字机效果逐字显示，不卡顿
- [ ] 回答结束后，下方显示来源引用（蓝色边框卡片，包含文档名和相似度）

---

## 4. 技术决策

- 混合检索权重：向量 0.4 + 关键词 0.6（AI-SPEC 规定，不可改）
- 对话历史存储在内存 Map 中，conversationId 作为 key
- SSE 流式格式：`data: {"chunk": "...", "done": false}`，结束标记 `done: true`
