# PRD.md — HR AI Assistant 产品需求文档

> 本文档定义"做什么"和"不做什么"。所有功能实现必须能在本文档中找到依据。

---

## 1. 产品愿景

### 1.1 一句话描述
基于 RAG 的企业内部 HR 知识问答系统，让员工通过自然语言快速查询公司制度、政策和流程。

### 1.2 目标用户
- **企业员工**：查询年假、报销、考勤等公开政策
- **HR 专员**：上传/更新制度文档，减少重复性咨询

### 1.3 核心价值
- **即时回答**：无需翻阅手册，直接提问即可获得答案
- **有据可查**：每个回答标注来源文档，确保信息准确可信
- **持续学习**：上传新的 Markdown 文档后，系统自动解析、分块、生成 Embedding 并重建索引

## 3. 非功能需求

### 3.1 性能
- 单次问答端到端响应时间 < 30 秒（本地 7B 模型，CPU 推理）
- 流式首字延迟 < 8 秒（含检索 + Prompt 组装 + LLM 首 Token）
- 支持并发：单机内存模型，建议 ≤ 5 并发
- 文档上传后索引重建：≤ 10 秒（单文件 < 100KB）

> **硬件说明**：以上指标基于 `qwen2.5:7b` 在 16GB RAM + Apple Silicon M2+ 或同等算力环境下的实测估算。纯 CPU 推理（无 GPU 加速）场景下，首字延迟可能达到 10-15 秒，属正常范围。

### 3.2 可用性
- 后端启动时自动加载 `docs/hr-documents/` 目录下的 Markdown 文档并建立索引
- 上传新文档后，无需重启服务，自动触发增量索引
- 登录态持久化：LocalStorage 存储 JWT

### 3.3 可扩展性
- LLM / Embedding / VectorStore 均为接口抽象，支持一键切换

### 3.4 安全与隐私
- 登录使用 JWT，内存预置账号，不连接真实数据库
- AI 回答必须基于上传的 HR 文档，禁止引用外部知识
- 涉及个人隐私的查询必须拒绝回答
- 文件上传限制：仅接受 `.md` 文件，单文件 ≤ 1MB

## 4. 明确不做（Out of Scope）

- ❌ 真实用户注册/密码找回/邮箱验证
- ❌ 连接真实数据库（MySQL/PostgreSQL/MongoDB/Redis）
- ❌ PDF/Word/Excel 文件上传解析（仅支持 Markdown）
- ❌ 多租户/多企业/多部门隔离
- ❌ 复杂管理后台（权限管理、数据报表）
- ❌ 生产级监控、日志收集、报警系统
- ❌ 多语言国际化
- ❌ 模型微调/训练

## 5. 用户旅程

### 5.1 员工查询年假
1. 打开系统，看到登录页（Theme 跟随系统偏好）
2. 输入预置账号（`employee / 123456`）登录
3. 进入聊天界面，看到热门问题推荐（"年假怎么请？"）
4. 点击或输入问题
5. 系统显示"正在检索文档..."，然后流式输出答案
6. 答案下方显示来源：《年假制度》第 2 章，相似度 0.89
7. 员工追问："那病假呢？"
8. 系统承接上文，回答病假政策并引用来源

### 5.2 HR 专员上传新制度
1. 使用 HR 账号登录（`hr / 123456`）
2. 切换到"文档"标签页
3. 点击"上传文档"按钮，选择新的 `.md` 文件
4. 系统提示"上传成功，正在建立索引..."
5. 索引完成后，文档出现在文档列表中
6. 立即可以针对新文档提问

## 6. 成功指标

| 指标 | 目标值 | 验证方式 |
|------|--------|---------|
| 回答准确率 | > 80% | 基于已知文档的 20 个测试问题 |
| 幻觉控制（拒绝率） | > 90% | 对 10 个文档外问题，系统应明确拒绝 |
| 流式首字延迟 | < 2s | 本地 Ollama 直接测量 |
| 文档上传后可用时间 | < 10s | 上传后提问，能基于新文档回答 |
| 代码可维护性 | — | 模块边界清晰，接口可替换 |
